Initial Setup

Getting started with Claptrap is a breeze. Simply open up a terminal and run the jar with java -jar claptrap.jar. To start collecting emails point your application to use Claptrap's SMTP server.
To see a list of all the available runtime arguments use the flag -h or --help with the above command.
Please note Claptrap requires Java JRE 8. 

			
				SMTP Settings for Your Application
				Transport Type: SMTP
				Host: <IP or Hostname of Server Hosting Claptrap>
				Post: 2525 is the default (this can be changed with above runtime arguments)
				Authentication: False

				Email Username: anythingYouWant@yourChoice.someTLD
				Password: thisDoesNotMatter

Please note that the second portion of the sender email address (i.e. everything after the @ symbol) will be used as the server name.

Usage

Using Claptrap is just as easy. Point your browser to the hostname and http server port shown in the console (defaults to 4567).
If no emails have been received by Claptrap, the server selector will not be present.

Once the first email arrives a server select menu will appear.
Choose the server that has the same name was the first half of the username that you set earlier.
Click the reload button and the emails will appear.

To view the full email click the view button. The full message will appear in a new tab / window.
To delete a email click the delete button.
To delete all emails for the currently selected server, click the delete all button, located in the top navbar.
To delete all emails for the currently selected server in style, enter the Konami Code.

Advanced Hooplah

You can force Claptrap to default to a certain server with a query string param.
Simply do the following: 

			
				http://claptrapHost:clapTrapHTTPPort?server=ServerNameSetAfter@SymbolAbove
